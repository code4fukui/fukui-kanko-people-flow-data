# 福井県観光DX AIカメラオープンデータ

[English](README.md)

このリポジトリは、日本・福井県の主要観光地に設置されたAIカメラから集計されたオープンデータを公開しています。データは定期的に収集・処理・公開され、公的な利用や可視化に活用できます。

## 可視化

人々の流れデータを可視化するウェブアプリケーションが利用可能です。

- **[可視化アプリ](https://code4fukui.github.io/fukui-kanko-people-flow-visualization/)**
- **[可視化アプリのソースコード](https://github.com/code4fukui/fukui-kanko-people-flow-visualization)**

## カメラ設置場所

福井県の以下の主要観光地にAIカメラが設置されています。

- 福井駅東口
- 東尋坊
- レインボーライン山頂公園（駐車場1・2）

## ディレクトリ構成

- `full/`: 完全な集計済みCSVデータ（場所と検出対象ごとに分離）
- `monthly/`: 日単位で集計されたデータ（`location/target/year/month`で構成）
- `daily/`: 時単位で集計されたデータ（`location/target/year/month/day`で構成）
- `hourly/`: 5分間隔で集計されたデータ（`location/target/year/month/day/hour`で構成）
- `tools/`: データ集計・処理用のTypeScriptスクリプト（Denoで実行）

## データ処理ツール

`tools/`ディレクトリ内のスクリプト（Denoで実行）:

- `aggregate-day.deno.ts`, `aggregate-hour.deno.ts`, `aggregate5mins.deno.ts`: ログCSVデータを日単位・時単位・5分単位に集計
- `csv2sqlite.deno.ts`: CSVデータをSQLiteデータベースに変換
- `check-csv.deno.ts`: CSVファイルのエラーや警告を検証
- `escape-age-data.deno.ts`, `escape-movement-data.deno.ts`: 年齢・移動データのフォーマットを修正
- `license-plate-aggregation.deno.ts`: 駐車場のナンバープレートデータを集計
- `thin-out-to-one-second-interval.deno.ts`: 移動データを1秒単位に間引く

## データ形式

CSVファイルには3種類の検出対象の集計値と属性が含まれます。

- **`Person`**: 一般的な人物検出
- **`Face`**: 年齢や性別などの推定属性を伴う顔検出
- **`LicensePlate`**: 車両のナンバープレート検出（都道府県・車両種別を推定）

## データの利用可能性と制限

### 一般的な運用

- **カメラ稼働時間**: すべてのカメラは24時間365日稼働しますが、メンテナンスや設備故障時は除きます。
- **ゼロカウント**: データの`0`はそのインターバルでカメラの視野内に検出対象がなかったことを示します。カメラがオフラインだったとは限りません。

### 検出に関する注意点

- **顔検出**: 24時間稼働しますが、早朝（午前8時前）と夜遅く（午後7時後）は検出数が大幅に減少します。理由は以下の通りです:
  - 外部照明条件が顔認証精度に大きく影響します。
  - 福井駅東口では営業時間外にセキュリティシャッターが降りており、カメラの視野が制限されます。
  - ローカル光条件下では検出アルゴリズムの感度が自然に低下します。
  - これらの時間帯の実際の来訪者数が少ないためです。
  - 検出数が少ない場合でも、カメラは顔を録画・処理しています。
- **人物検出**: すべての時間帯で一貫した検出能力を持ち、24時間分析には顔検出よりも信頼性が高いです。
- **ナンバープレート検出**: カメラシステムが稼働している間は24時間利用可能です。

### データの特性

- すべての検出タイプは24時間365日連続稼働します（メンテナンスや既知の停止時を除く）。
- 顔検出は1日を通じて自然な変動があります:
  - 非常に低い検出数: 午前0時～午前8時（福井駅では平均<100検出/時間）。
  - 最大検出数: 午前11時～午後5時（福井駅では平均>12,000検出/時間）。
  - 低い検出数: 午後7時～午後11時（平均<500検出/時間）。
- 人物検出はすべての時間帯で比較的一貫した24時間カバレッジを持ち、合理的な検出数を維持します。
- データは営業時間や観光シーズンのピーク活動を反映した実際の来訪者パターンを示します。
- 時間単位データは5分間隔で提供され、詳細な時間分析に活用できます。

## 既知の問題とデータギャップ

- **データ収集開始**: データ収集は**2024年12月20日**に開始されました。この日以前のデータはありません。
- **システム停止**: **2025年9月26日～28日**にすべての場所で完全なシステム停止が発生しました。
- **設備故障**: **レインボーライン駐車場1**のナンバープレート検出カメラに運用上の問題が発生しました（具体的な日付は文書化待ち）。
